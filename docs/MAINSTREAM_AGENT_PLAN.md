# Rcode 主流 Agent 产品化计划

## 目标

Rcode 的方向是主流 coding agent 产品，而不是普通聊天应用。产品参照对象包括 Codex、Claude Code、Cursor Agent 和 GitHub Copilot Coding Agent。

## 第一阶段：定位与接入

- 明确 Rcode 是 coding agent。
- MiMo 作为默认通用免费模型入口。
- 支持用户自定义模型服务。
- 支持 OpenAI-compatible、OpenRouter、自定义 Base URL 等通用协议。
- 在设置中提供 provider 管理能力。

## 第二阶段：开发闭环

- 项目上下文识别。
- 文件读取与编辑。
- 文件差异审查。
- 任务过程展示。
- 权限审批与风险提示。
- 会话与任务记录。

## 第三阶段：主流 agent 能力补齐

- 项目搜索。
- 代码结构理解。
- 任务规划。
- 测试与构建结果展示。
- Git 工作区状态展示。
- Patch 接受、撤销和回滚。
- MCP 或插件式工具扩展。

## 第四阶段：桌面产品化

- 完善设置页。
- 完善模型管理。
- 完善桌面端打包。
- 增加新手引导。
- 增加默认模板和常用任务入口。

## 不做什么

- 不把 Rcode 定位成普通聊天壳子。
- 不把 Rcode 定位成某一个模型的专属客户端。
- 不把 MiMo 当成唯一核心卖点。

## 关键表述

Rcode 的地位是对标 Codex 等市面主流 agent。
