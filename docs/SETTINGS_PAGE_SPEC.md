# Rcode 设置页规格

设置页是 Rcode 对齐主流 agent 产品的关键入口。它不只是偏好设置，而是模型、权限、工具和本地 agent 行为的控制中心。

## 1. 模型设置

### 目标

让用户可以使用默认通用免费模型，也可以接入自己的模型服务。

### 字段

- Provider 显示名称
- 协议类型
- 服务地址
- 默认模型
- 备用模型
- 是否支持流式输出
- 是否支持工具调用
- 是否设为当前默认

### 默认项

- Xiaomi MiMo：默认通用免费模型入口
- OpenAI-compatible：通用兼容协议
- OpenRouter：聚合模型协议
- Custom：用户自定义兼容服务

## 2. Agent 模式

### 模式

- 快速：更少探索，更快响应
- 标准：兼顾速度与可靠性
- 深度：更适合复杂项目任务

### 后续可扩展

- 规划优先
- 测试优先
- 审查优先
- 重构优先

## 3. 权限设置

### 模式

- 请求批准
- 替我审批
- 完全访问

### 细化项

- 文件读取策略
- 文件编辑策略
- 网络访问策略
- 本地任务策略
- 项目外路径策略
- 高风险操作处理方式

## 4. 工具设置

### 当前工具

- 文件读取
- 文件编辑
- 网页获取
- 本地任务执行

### 后续工具

- 项目搜索
- Git 状态
- 测试结果
- 构建结果
- 浏览器
- Figma
- 数据库
- 文档源

## 5. 产品定位露出

设置页顶部应明确显示：

Rcode 是对标 Codex 等市面主流 agent 的本地 coding agent。

MiMo 是默认通用免费模型入口，不是 Rcode 的唯一模型，也不是产品定位上限。

## 6. 推荐信息架构

- 常规
- 模型
- Agent
- 权限
- 工具
- 项目
- 关于 Rcode

## 7. 最小可交付版本

第一版设置页只需要完成：

- 展示当前 provider
- 展示当前模型
- 展示默认权限模式
- 展示 Rcode 产品定位
- 预留添加 provider 的入口
