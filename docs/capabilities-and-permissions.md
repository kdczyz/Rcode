# 能力与权限基线

Rcode 使用“工作区边界 + 工具级审批 + 审计”的分层策略。它不是管理员权限替代品，也不宣称提供 OS 级沙箱。

## 能力层

| 能力 | 实现 | 默认策略 |
| --- | --- | --- |
| 受控通用终端 | `run_shell`、长期进程会话、安全环境变量白名单 | 工作区内测试、构建、格式化和 Lint 自动允许 |
| 浏览器自动化与截图 | `native-devtools-mcp` 的 CDP、Accessibility、截图工具 | 页面观察与截图允许，点击、输入和窗口修改需审批 |
| IDE/诊断 | `project_diagnostics` 调用项目已有的 `typecheck`、`lint`、`test`、`build` 脚本 | 自动允许，不自动安装诊断器 |
| 文档和源码搜索 | `search_text`、`list_files`、`inspect_tree`、`web_fetch` | 项目内搜索允许，联网获取需审批 |
| Docker 与数据库 | 结构化 `docker_compose`、`sqlite_query` | 状态、日志、只读 SQL 允许；容器和数据修改需审批 |
| 云平台与 Xcode | 受控终端或按需 MCP server | 部署、云资源修改、归档导出均需审批；生产操作逐次审批 |
| 密钥注入 | `secretRefs` 引用 `[secrets].allowed_env` 中的本机变量 | 不把值放入工具参数；每次注入需审批；stdout/stderr 自动脱敏 |

## 强制策略

- 自动允许：项目内文件读写、搜索、测试、构建、格式化、Lint 和只读诊断。
- 执行前展示：安装或升级依赖、联网请求、数据库迁移、容器修改。
- 必须确认：Git 分支/暂存/提交/推送、部署、云资源修改。
- 逐次确认：删除数据、生产环境操作、`sudo` / `doas` 等提权命令。批准只对当前工具调用有效。
- 默认禁止：工作区外个人文件、`.env*`、SSH 目录和私钥、macOS 钥匙串、浏览器登录数据库，以及可能转储环境变量的命令。
- 无论权限模式如何，托管的强制拒绝与强制审批规则都不能被项目规则降级。

## 安全密钥注入

1. 在 `config/agent.toml` 的 `[secrets].allowed_env` 中显式允许变量名。
2. 在本机 `.env.local` 或启动环境中设置变量值；不要把值写入项目配置、提示词或工具参数。
3. Agent 调用 `run_shell` / `start_process` 时只传 `secretRefs: ["变量名"]`。
4. UI 展示变量名并请求本次批准。执行器只向子进程注入所选变量，并将输出中的实际值替换为 `[REDACTED]`。

注入不能阻止第三方程序把凭据发送到其目标服务，因此密钥仍应遵循最小权限、短有效期和可撤销原则。

## 边界

当前便携执行器依赖路径规范化、命令分析、收窄后的子进程环境和审批流程；它不是 macOS App Sandbox、Linux namespace/seccomp 或虚拟机隔离。生产环境建议再叠加容器或专用执行主机。
