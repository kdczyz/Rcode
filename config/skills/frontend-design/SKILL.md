# Frontend Design

Build the actual working interface around one clear visual and interaction thesis.

## Workflow

1. Inspect the existing design system, component patterns, and responsive constraints.
2. Define the screen's primary job, dominant region, supporting context, and main action.
3. Establish hierarchy with typography, spacing, alignment, and contrast before decoration.
4. Implement responsive states, focus and hover states, loading, empty, and error behavior.
5. Verify the real rendered interface at desktop and narrow widths.

## Quality Bar

- Prefer calm, dense, readable product UI over decorative card mosaics.
- Reuse existing tokens and icons.
- Keep controls stable and labels scannable.
- Use motion only to clarify state or hierarchy.
- Preserve keyboard access, semantic labels, and readable contrast.
